GitHub & Git

.gitignore

It's not without reason to say that when uploading files to your repository, there may be files that you would prefer not to upload, whether they are files with private information or perhaps they are just unnecessary files that have no business being in your repository. While you clearly don't have to upload them, you will continue to see these files pop up again and again in you list of untracked files. This will become a bit of an annoyance as time goes by, especially if those files begin to pile up and it becomes more difficult to upload the files you want to. Luckily, there is a method to solve this problem.

To take these files off your tracked files list, you can either take them out of the repository, or you can create a .gitignore file. This files is a special file recognised by Git that will allow it to ignore those files. Essentially, it's just a list of files and folders fir Git to ignore. You can create one through probably any IDE editor or through the Git terminal. Using an IDE, just create a new file in the folder with the cloned repository and name it ".gitignore". Within the terminal, you navigate to the same folder in your repository and enter "touch .gitignore". That's all there is to make one, but you will still need to add files and or folders to it.

You can then open it in the editor and add the files and folders one by one, line by line. If you wanted to ignore "example.txt", you would enter it exactly as it's spelled, including the file extension. If you wanted to ignore the folder "example" which is in the New Folder in your repository, you would enter "New Folder/example/*". This will indicate that everything within that directory and the folder itself will be ignored.

There's perhaps an even easier method where you could go to THIS page in GitHub where there are numerous .gitignore files available to download for major frameworks, some IDEs like Visual Studio Code, and many other programs. The files listed in these are suggested by veteran web developers and they are being updated fairly often.
