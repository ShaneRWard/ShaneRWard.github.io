UTF-8

Binary code is the basic language that computers use for transmitting data either between components or to other computers. It's also used in data storage. It's a base 2 system which means it makes up codes using combinations of two numbers, 0 and 1. The ASCII encoding system utilizes binary code by making sequences of 7 1's and 0's called bytes (each individual digit is called a bit), giving a maximum of 128 possible combinations and are used for characters like letters or numbers.

The UTF-8 is a clever way of using that older ASCII system as a base and expanding it greatly, making it usable as a global standard. The larger character possibilities would have made the sequences much longer, but less efficient and incompatible with older systems. But UTF-8 found a way to not only remain compatible with older systems using the 7 digit bytes, but also eliminate the waste in excess digits making it highly effiecient.

The need for UTF-8 may or may not have been foreseen when ASCII was made. UTF-8 was an incredible development and I'm curious how long it will carry us before it becomes obsolete and another system is needed to keep up with our evolving technology.
