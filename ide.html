Integrated Development Environments

HTML and CSS can be encoded using programs as basic as notepad in Windows. However, there are programs available that will make the process much, much faster and easier. These are called Integrated Development Environments, or IDEs for short. Most of these are available online for free, though not all IDEs are available free of charge. Below, we will go over a few of these IDEs, and discuss their advantages and disadvantages.

VS Code

Visual Studio Code is a fairly new player in the game, but it has very quickly made a name for itself. Though relatively small, VS Code is a fantastic code editor. It is at the top of many lists for favorite IDEs.

  1. 1. Available for free online.
  2. 2. Has a smart autocomplete system, IntelliSense, that does an incredible job of speeding up coding.
  3. 3. Has a broad variety of extensions to improve and customize coding experience.
  4. 4. Has a built-in debug mode.
  5. 5. Has a terminal with built-in Git commands.

Atom

Atom is a veteran favorite among IDEs made by the devs at GitHub, an open source community.

  1. 1. Made by GitHub, and works directly with GitHub and Git.
  2. 2. Highly customizable with lots of extensions.
  3. 3. Toutes how "hackable" the program is for ultimate customization.
  4. 4. Free online.
  5. 5. Thousands of open source packages that add features and functionality to Atom. Can create your own in Atom.

Ruby

Ruby is another favorite among IDEs. Made by JetBrains, another powerhouse in web development.

  1. 1. Unfortunately, NOT free. Caries a hefty price tag.
  2. 2. Toutes best in class coding assistance.
  3. 3. Has plugin support for testing of Node.js, Dart, Angular js, and JavaScript frameworks.
  4. 4. Has smart code completion functionality.
  5. 5. Has a unified UI for working with many version control systems such as Git, GitHUB, Mercurial, SVN, and Perforce.

Web Storm

Web Storm is another IDE by JetBrains, another powerhouse in web development.

  1. 1. Like Ruby, NOT free. Caries a hefty price tag.
  2. 2. Has good debugging tools.
  3. 3. Has a unified UI for working with many version control systems such as Git, GitHUB, Mercurial, SVN, and Perforce.
  4. 4. Has a local history where you can roll back to previous versions for a file or directory.
  5. 5. Has lots of customization tools
