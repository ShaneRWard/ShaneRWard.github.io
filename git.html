GitHub & Git

Git

Git is an open source system used for version control. It is not a part of GitHub, though for our purposes we will be using it to upload files to GitHub. It's program that uses CLI and similar commands. It has integrated several diffent kinds of Command Line Interface so they can all be used for its purposes. When paired with resources like GitHub, it makes a great tool for collaboration between developers. It helps them keep on the same page by allowing everyone access to the same code. It can be used in making branching versions of code which can be merged with the master version when needed. It's an essential tool for developers. Git is available to download for free HERE.

Git uses a staging process in it's workflow that is important. It can be described as having three different tiers; the working directory, the staging area, and the repository. Files you work on are part of the working directory. Files can then be added to the staging area for review or modified before committing them to the repository. Modified files when committed, are expressed as a set of changes rather than a whole file to replace the older version.

There are a couple different programs associated with Git, but for our purposes, we are going to focus on the terminal program, Git Bash. Any time throughout this site that "terminal" is mentioned, it is refering to Git Bash. Some IDEs can be set up to use this terminal, which can be handy, but not entirely necessary. Here we will go over several basic but essential commands. Many of these are mentioned in other sections, but this shall work as a reference guide.

NOTE: Every git command starts with "git"; example: git add example.txt. Parenthesis are placeholders and are not used unless otherwise specified.

Command Description
git cofig --global user.name "username" This will configure Git to assign an Author associated with your commits. After installing Git, this may be required the first time you use it.
git config --global user.email Like the username, this will assign an email address associated with your commits. May be required the first time you use Git.
git init Creates a new local repository.
git remote add origin (url) Connects to a remote repository. Enter the URL of the remote server. Do not use parenthesis.
git pull Pulls changes from your remote repository and merges them with your local repository. If changes happen to the remote repository, you will have to pull them to local repository to resolve conflicts.
git status List all files that need to be added, changed files, and files that need to be committed. Files ready to be committed are shown in green while changed files and untracked files are shown in red.
git add (filename) Adds specified file to staging area.
  • git add --all
  • git add *
  • git add -A
 Adds all untracked and modified files to staging area.
git commit -m "message" Commits staged files to repository with message.
git push Pushed committed files on current branch to remote repository.
git log Provides a log of commits.
git branch Lists all branches within repository and which branch is currently active.
git branch (name) Creates a new branch with the specified name.
git checkout (name) Switches from one branch to another.
git checkout -b (name) Creates a new branch with specified name and switches to it.
git merge (branchname) Merge two branches into one. Must be checked out under another branch.
