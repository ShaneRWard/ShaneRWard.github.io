Repos

A repository, or also known as a "repo", is as the name suggests, a space where you can publish work on GitHub. This can be a menagerie of things such as text files, web pages, and coding files of all sorts. this is an extremely useful resourse and has become a popular tool for developers across the globe. It's a playground of open source projects and many popular programs, frameworks, and other things are continually developed and updated using GitHub.

Creating a repository is fairly simple and there's a couple ways to go about it. The first thing you need to do is to create a repo on the GitHub website itself. After you sign up and get your account, you can start putting together your repo.

From your dashboard on the website, you can go to the left panel and next to "Repositories" there is a green "New" button. This is also found on your profile page. On the next page is where you will name and create your repository. Here you can select who the owner of the repository shall be (this will only be you until you join an organization), name the repository and give an optional description of your repo, select if the repo shall be private or public, you can add a READ ME file if you want to provide information to others who may see your repository, add a .gitignore file from a list of various kinds, and add a license which we will not cover in this guide.

After you hit create, you are sent to the repository setup page. Here is where you can clone the repository to your machine and you can work on it locally.
