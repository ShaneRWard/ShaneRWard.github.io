Cloning Repositories

Once you have created a repository on github, you will be sent to a setup page that provides instructions on how to copy or clone the new repository from GitHub to your computer so you can work on it locally. On this page there will be a URL that you will need. Either copy it or write it down.

The first thing you should do is create a folder with a name matching that of the repository. This is where you're going to clone the repo. From here, you will need to use the Git terminal to navigate to that new folder.

Once you are there, begin by entering the following command: git init. This will initialize the repository and create hidden files within that directory. Then, enter the following: echo "# Hello-World" >> README.md. This will make the README file in the repo folder. You're going to first add the readme file to the tracked files by entering git add README.md, then commit it by entering git commit -m "First commit". The readme file is staged and ready to be pushed, but first it needs to know where it's going to be pushed to.

Here is where that URL comes in. In the terminal, enter git remote add origin (URL). *Note do not include the brackets, just the URL* Almost done! Now comes the final "push". Enter git push -u origin master. Finally, enter your login credentials when prompted.

That's it, you're done cloning! If you check the new GitHub repository. You will see the README.md file loaded.

One final note, if you've already started a project locally and wish to make it into a repository, the process is essentially the same, just make sure the name of the folder it's in matches the name of the repository, navigate to that folder in the Git terminal, do a git init, git add *, git commit, git remote add origin (URL), and a git push -u origin master.
