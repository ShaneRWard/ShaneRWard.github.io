Branches and Master Branch

Branching is an imporant part of Git, GitHub, and version control. It allows teams to work in parallel to each other, to experiment and proper implementation of features. It helps to eliminate risk. As a general rule, no branch should ever be permanent. In fact, no branch should last any longer than it needs to be. Provided here is a guide to using branching using Git.

First and foremost, let's start with creating a branch. Say a problem arises in your code. It may not be prudent to simply jump into the code and try to fix it directly. You would want to branch out, create a fix, then merge back into the main branch. Click through the tabs below to learn more about branching.



  • Branching
  • Checkout Branches
  • Merge Branches

Take the example given above. An major bug has been found in the code and it needs to be fixed quickly. To create a new branch, we must go into Git and enter the following command: git brach (branchname). Take a look at the example below.

git branch hotFix

Now we have a new branch. We can work on this hotFix to solve an issue in our code. Once we have completed work on the fix, however, we cannot simply commit those changes because Git thinks we are still on the Master branch. We need to switch to the hotFix branch. Click over to the next tab to learn what to do next.

To move to the hotFix branch we need to check it out.

git checkout hotFix

Now we have switched from the Master branch to the hotFix branch. Now we can commit our work.

git commit -m"hotFix added.

It may be prudent to commit the branch as soon as you create it. You can do this in one line using the following:

git checkout -b hotFix

Now that our fix has been made, we have committed it, but it's not yet a part of the main code. Instead of an inordinate amount of time copying and pasting and checking, we can cleanly merge the two branches.

First we checkout the Master branch, then we merge it with the hotFix branch.

git checkout master; git merge hotFix

*Note, the example above displays how two commands can be entered on one line separated by a ";".*
