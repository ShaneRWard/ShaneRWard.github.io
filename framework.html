What is a Framework?

A framework is a collection of CSS, Javascript, icon libraries, and files of standardized code used for building websites. They can be very useful tools when it comes to web development. They help to build responsive and dynamic pages that should appeal to users. They can save a lot of time and effort once you become familiar with them. They do, however, have their drawbacks. As said before, they are libraries of code, which means they will have a lot of code that you don't need. That excess code can make your website bulky and slow. This isn't always the case as there are some minimalistic frameworks designed to reduce excess data and the benefits of using frameworks usually outweight the risks of using them. Below we will go over some of the more popular frameworks.

Bootstrap

You may not have heard of Bootstrap if you're new to web development, but then you probably didn't know what a framework is either. Bootstrap is an open source framework and is probably the most popular of frameworks at this time. It's been out since 2011, so it's had plenty of time to build a following and it's updated continually. It has a lot of the latest and greatest tools for web building. As such, it's also one of the bulkier frameworks out there because many of its styles are not used very often. It comes with the Sass CSS preprocessor (This is a program that creates CSS from it's syntax. They often add features that CSS can't usually do on it's own). It provides a nice variety of elements like nav bars, form styles, list styles, wells, grids, and more. You can find it here.

Pure.CSS

As mentioned before, there are a few minimalist frameworks that have a very light load. Pure is one of those frameworks. As the name suggests, it is purely CSS. No Javascript, no jQuery, no extra bells and whistles. It's so thin that it only makes up 400 lines of code. This makes it very light, 3.8kb when fully compressed and minified, yet it's still very responsive. Like others, it has nice grids, buttons, tables, and menus. It's light design makes it geared toward use with mobile devices which may process slower than a PC. It's skeletal design may make it easier for inexperienced developers, but like Pure.CSS, it lacks a the broad selection of modules other frameworks would have and any Javascript. You can find Pure.CSS Here.

Skeleton

Speaking of skeletal designs, Skeleton is another bare bones framework meant for quick and basic sites. It's so thin, it makes up only 400 lines of code. Described as more of a boiler plate, it's intended to be a starting point from which to build up from. As such, it may be be suitable for newer developers. Skeleton can be found Here.

Semantic-UI

If you're one looking for lots of tools to work with and are very new to development, Semantic-UI is a great framework. Its language is simple, self explanitory, and designed to make web building easier for inexperienced developers. One of it's more notable features is that it has a massive amount of third party libraries to work from, giving you anything and everything you need to make the website you want. What's better is that these file sizes are small and you can add only what you need. This is very handy considering Semantic-UI is very large compared to other frameworks. Semantic-UI is frequently updated and it updates your version automatically so you don't have to worry about it. If this sounds like the right framework for you, go to it webpage Here.
